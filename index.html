5 Ways of Thinking Like a Programmer

  • Procedural Thinking - creating prefectly clear and unambiguous instructions for a computer to follow.
  • Abstract Thinking - finding the similarity or generality amongst seemingly different things.
  • Systematic Debugging - relentlessly identifying the cause of a computer program to not to work. When a program doesn't work, it's because there is a mistake/bug somewhere in the computer code.
  • Technological Empathy - comes in many forms. For example, computer empathy is the ability to understand what a computer is, how it works, and what its good and bad at doing.
  • Systems Thinking - happens when you break a big problem down into smaller pieces.

Session 1: Introduction to "Serious" Programming

  • A Computer is a machine that can be programmed to do whatever you want it to do.
  • A Computer Program is a precise sequence of steps that tells the computer what to do.
  • When you divide an integer (number without a decimal) by another integer, Python does something called integer division which means it ignores the decimal part of the number. To force Python to give you an answer including a decimal, you need to make one of the numbers into a decimal point after it. For example 5/2.0, gives 2.5, and so does 5.0/2.
  • A Programming Language is what programmers use to tell computers what to do. Python, Cobol, and Fortran are examples of programming languages.
  • Python is a programming language that utilizes a Python Intepreter to follow the instructions of your code in a way that the computer understands.
  • Grammer is very important when writing code. You must use a set of instructions that is correct, precise and one that the computer understands.

Session 2: Variables and Strings

  • Variable - a name that refers to a value. A variable name in Python can only contain letters, numbers, and underscores. However, the 1st character cannot be a number. Variables are case sensitive. For example, username is not the same as userName.
  • The = sign is an assignment statement and it is used to assign a value to a variable. It means we are assigning the value on the right side of the = sign to the variable on the left. Example myAge = 25.
  • A String is a sequence of characters between single or double quotes. For example: 'I am a string' or "I am a string".

Session 3: Inputs->Functions->Output

  • Functions takes input, do something with it, and then produces output.
  • A function begins with def defines/names a function. For example: def myName(x):
  • def tells the program that the indented code from the next line onwards is part of the function.
  • The line must end with a : colon
  • The function name is followed by an open parentheses, followed by the value of the required parameters, followed by a closed parentheses.
  • Every line within a function must be indented.
  • There can be more than one return statements in a function.
  • Once the function executes a return statement, the function will exit.
  • Functions that produce an output, ends with a return statement.
  • To Display a function on the screen, you must include the print statement before you call your function.
  • Some of the advantages of using Functions are as follows:
    • Reduces Debugging time by making it easier to locate an mistake within the code.
    • Reduces coding time by allowing you to write the routine once, and code statements that call it.
    • Improved readability of the program.
    • Reduces duplication of code.
    • Aids in breaking down complex problems into simplier pieces.
    • Allows reuse of code.

Session 4: Control Flow & Loops: If and While / Debugging

  • The equality comparison is made by using '==' instead of '=' because the '=' means assignment. For example: 7 * 3 == 21
  • Loops are a way to do things over and over.
  • If statements evaluate if a certain condition is met, and to perform the appropriate action based on the result of the evaluation.
  • While loops repeatedly executes instructions insided the loop while a certain condition is true.
  • Debugging Strategy
    • Examine error messages when programs crash. The last line of Python Tracebacks will tell you what line it crashed on, what file it was running, and how it got there.
    • Work from example code. If you modified code doesn't work, comment it out and step-by-step modify the example code until it does what you want.
    • Makes sure examples work.
    • Check/print intermediate results. Add print statements in your code to make sure that your program is doing what you want it to do.
    • Keep and compare old versions. When troublshooting your code, save it before you add more code. This will give you something to go back to if you introduce to many new bugs.

Session 5: Structured Data: Lists & For Loops / How to Solve a Problem

  • Lists are a sequence of anything.
  • For Loops executes a block of code repeatedly until the condition in the for statement is no longer valid.
  • Indexing allows you to return the position where the value is found in a list. Otherwise, it produces an error.
  • How to Solve a Problem:
    • Make sure you understand the problem. What are your inputs? What are your outputs?
    • Work out examples. Write pseudo-code.
    • Write small bits of code, test them, know what they do.
